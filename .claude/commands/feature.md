# /feature — 기능 전체 워크플로우

## 역할
기능 구현의 전 과정을 단계별로 안내한다.
각 단계 사이에 반드시 사용자 확인을 받는다.
사용자 확인 없이 다음 단계로 자동 진행하면 안 된다.

## 입력
$ARGUMENTS — 구현할 기능 설명

## 워크플로우

```
/plan → 승인 → /impl → 테스트 → /review → 승인 → /commit → /pr
```

---

### STEP 1: 계획 수립 (/plan)
$ARGUMENTS를 기반으로 /plan을 실행한다.
계획을 사용자에게 보여주고 다음 메시지와 함께 대기한다.

```
계획 수립이 완료됐습니다.
검토 후 "승인"이라고 답하시면 구현을 시작합니다.
수정이 필요하면 수정 내용을 알려주세요.
```

---

### STEP 2: 구현 (/impl)
사용자가 승인하면 /impl을 실행한다.
구현 완료 후 자동으로 STEP 3으로 진행한다.

---

### STEP 3: 테스트 실행
```bash
./gradlew test
```

**테스트 성공 시**
STEP 4로 진행 안내:
```
테스트가 모두 통과했습니다.
/review를 실행합니다.
```

**테스트 실패 시 처리 (최대 3회)**
1. 실패 원인 분석 및 수정 시도
2. `./gradlew test` 재실행
3. 3회 연속 실패 시 즉시 중단하고 사용자에게 보고:
   ```
   테스트가 3회 연속 실패했습니다.
   실패 로그:
   <로그 내용>

   수동 확인이 필요합니다. 원인을 함께 분석해 드릴까요?
   ```

---

### STEP 4: 리뷰 (/review)
/review를 실행한다.
결과를 사용자에게 보여주고 다음 메시지와 함께 대기한다.

```
리뷰가 완료됐습니다.
FAIL 항목이 있다면 수정 후 "승인"이라고 답하시면 커밋을 진행합니다.
FAIL 항목이 없다면 바로 "승인"이라고 답해주세요.
```

---

### STEP 5: 커밋 (/commit)
사용자가 승인하면 /commit을 실행한다.
커밋 메시지를 제안하고 승인 대기한다.

---

### STEP 6: PR 초안 작성 (/pr)
커밋 완료 후 /pr을 실행한다.
PR 초안을 `.ai-workspace/pr.md`에 저장하고 사용자에게 보여준다.
push와 PR 생성 명령어를 안내하고 종료한다.

---

## 전체 진행 상황 표시
각 단계 시작 시 현재 위치를 표시한다.
```
[feature 워크플로우] STEP 2/6 — 구현 중 (/impl)
```

## 주의사항
- 사용자 확인 없이 단계를 건너뛰거나 자동 진행 금지
- 어느 단계에서든 사용자가 중단을 요청하면 즉시 중단
- push 자동 실행 절대 금지
- 보안 관련 파일(SecurityConfig, JwtAuthenticationFilter, JwtTokenProvider) 수정이 포함된 경우 STEP 2 시작 전 별도 경고
